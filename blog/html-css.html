    Block, inline, and inline-block:

      BLOCK Elements - Key Points
  • Block level elements always stack on top of eachother, even if they have room to go next to eachother, they will start on a new line underneath.
  • Their default width is 100%, see Fig. 1:

    • Fig. 1

  • Even if you change their width to 33%, or 25%, they still won't sit next to eachother, they'll always stack. See fig. 2:

    • Fig. 2



      IN-LINE Elements - Key Points
  • Inline elements always sit within the block element they are in. They won't start a new line. See fig. 3 & fig. 4:

    • Fig. 3

    Fig. 4

  • You cannot change the height of the inline element, because it throws the rest of the block dimensions off.
  • But you can alter the font-size of the inline text, so it can be bigger, but it makes the surrounding inline content adjust the spacing to fit that new size. See fig. 5 & fig. 6:

    • Fig. 5

    Fig. 6



      IN-LINE BLOCK Element - Key Points
  • Inline-block allows you to set the height (top and bottom padding/margin) just like a block element, but they will also have the ability to sit next to eachother like an in-line element.
  • As we learnt with inline elements before, you cannot control the height or position of inline elements. As you can see with the button example of fig.7:

    • Fig. 7




  • But with "Inline-Block" you change your CSS to make it display: inline-block; which means you will be able to set the height and control it like a block element. See fig. 8 & fig. 9:
